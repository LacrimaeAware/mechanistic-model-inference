# Documentation update map

Keep the smallest set of documents current that preserves the project trail.

| Location | Update when |
|---|---|
| `README.md` | scope, status, or layout changes |
| `AGENTS.md` | conventions change |
| `docs/goals.md` | learning or project goals change |
| `docs/research_questions.md` | a question is added or sharpened |
| `docs/roadmap.md` | a phase status or direction changes |
| `docs/literature.md` | a source is gathered, verified, or summarized |
| `experiments/NN_*/*.md` | an experiment is added or rerun with results |
| `docs/update_map.md` | these conventions change |

Generated data under `data/` and outputs under `results/` are git-ignored and stay local. Use
descriptive experiment note names, not `README.md`, inside `experiments/`.
